# AVM-Spider

Spider 是 AVM 的状态管理内核。它彻底放弃了全局状态锁的范式，以**对象**为最小管理单位，类似现代数据库的行级锁。

核心设计采用 Chain-Block-Piece&Event 三层模型：

- **Space**：策略域，定义权限、生命周期和成本策略，不存储业务数据
- **Block**：数据对象，一致性策略（STRONG/BRANCH）的最小单位
- **Piece**：数据快照，只存数据内容，不存策略元数据
- **Event**：事件系统的根基，存储事件条件和行为。

不同对象的状态变更互不干扰，天然支持高并发。Spider 采用乐观执行、顺序锁写入的两阶段并发控制——冲突仅回滚受影响对象，而非整笔交易。所有状态变更生成 ZK 友好的证明结构，支持细粒度的对象级权限控制（R/W/Admin/Billing 四项独立权限 + 委托机制）。合约大小与计算量不再受 EVM 式限制。
